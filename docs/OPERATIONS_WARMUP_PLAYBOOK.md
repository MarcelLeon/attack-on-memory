# Operations + Warmup Playbook (Owner-light)

This playbook is designed so maintainers can run launch ops with minimal manual effort.

## Weekly cadence
- Mon: publish a technical insight post (architecture/governance)
- Wed: publish a benchmark/progress update
- Fri: publish a failure-analysis or roadmap update

## Daily 15-minute loop
1. Triage new issues/discussions
2. Label and prioritize
3. Reply to top 3 community questions
4. Convert repeated questions into FAQ updates

## Growth loop
- Convert high-signal feedback → issues
- Convert solved issues → release notes wins
- Convert release notes wins → social proof posts

## Guardrails
- Never over-claim benchmark results
- Always include reproducible commands for claims
- Be explicit about limitations
